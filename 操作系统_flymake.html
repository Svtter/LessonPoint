2 TODO 进程管理

2.1 进程的概念，特性

进程是对正在运行的程序的一种抽象，是资源分配和独立运行的基本单位。

2.2 进程的状态及其转换（具有挂起状态）

运行状态 - *阻塞状态*（静止阻塞，活动阻塞） - *就绪状态*（活动就绪，静止就绪）

  • 特别的，无法从阻塞状态直接转换成运行状态

2.3 PCB的作用，内容，组织方式

  1. 作用
    • 为了保存进程的各个状态，建立专用数据结构，即为PCB
  2. 内容
    • 进程标识，位置信息，状态信息，进程的优先级，进程现场保护区，资源清单，队列指针等
  3. 组织方式
    • 线性表
    • 链接表
    • 索引表

2.4 各种进程控制的原语（创建，终止，阻塞，唤醒，挂起）

原语指由若干条指令组成，不可被打断。

  1. 创建

    申请空间 - 查找PCB表 - 分配PCB空间 - 分配资源 - 信息填入PCB - PCB插入就绪队列

  2. 终止

    找到PCB - 没有，进入异常处理 - 有，撤销PCB链中该进程的名字以及所有的子进程

  3. 阻塞

    中断CPU - 将CPU状态存入PCB - 进程状态转变为阻塞 - 插入等待队列 - 执行调度程序

  4. 唤醒

    唤醒条件满足 - PCB状态设置就绪，脱离阻塞队列 - 插入就绪队列

2.5 线程的概念，进程与线程的联系与区别

  1. 概念

    线程是进程中的一个实体，是被系统独立调度和分配的基本单位

  2. 联系区别
    1. 调度

      传统操作系统中，拥有资源的基本单位和独立调度的基本单位都是进程，而在引入线程的操作系统中，则把线程 作为调度和分配的基本单位，而把进程作为拥有资源的基本单位，使传统进程的两个属性分开

    2. 并发性

      不仅进程之间可以并发执行，一个进程的多个线程也可以并发执行

    3. 拥有资源

      线程不拥有系统资源，但可以访问隶属进程的其他资源

    4. 系统开销

2.6 进程同步的概念

异步环境下并发执行的进程：

  1. 同步

    因直接制约而需要相互等待，相互合作，以达到各进程按相互协调的速度执行的过程，成为进程的同步

  2. 互斥

    因间接制约而导致交替执行的过程称为进程的互斥

2.7 临界资源，临界区的概念

某段时间内只能允许一个进程使用的资源称为临界资源，必须互斥执行的程序段称为某临界资源的临界区

  • 进入区
    • 临界区
  • 退出区

2.8 同步机制原则

  • 空闲让进
  • 忙则等待
  • 有限等待
  • 让权等待（不占用CPU资源），忙等待则占用CPU资源

2.9 同步与互斥的各种方法（软件，硬件，信号量（集））

  1. 硬件方式
    • 中断禁用
    • 专用机器指令
  2. 软件方式
    • Dekker算法，Peterson算法 无法实现让权等待
  3. 信号量, P/V操作

2.10 经典同步问题

  1. 生产者消费者问题
    1. 生产者
      1. 有无空位
      2. 锁资源
      3. 释放资源

    2. 消费者
      1. 有无可使用资源
      2. 锁资源

  2. 读者写者问题

2.12 各种调度算法（FCFS，SJF，RR，优先权法，多级反馈轮转）

  1. FCFS 先来先服务
  2. RR 时间片轮转
    • 分配时间片段给任何一个进程
  3. 短进程优先
    • 时间短的进程优先执行
  4. 高响应比优先
    • 公式： 优先权 = （等待时间 + 要求服务时间） / 要求服务时间
    • 需要计算响应比，会增加系统开销
  5. 优先级法

    进程的类型，运行时间，作业的优先级。

  6. 多级反馈队列调度算法
    1. 放在不同等级的队列上进行执行
    2. 在每一个队列上执行时间片轮转

2.13 死锁的概念，产生原因，必要条件，处理方法

  1. 概念

    某个进程提出资源申请的时候，有关进程无法在外力的协助下，永远分配不到必需的资源而无法继续运行。

  2. 产生原因
    1. 系统提供的资源不能满足每个进程的使用需求
    2. 在多道程序运行时，进程推进顺序不合法。
  3. 必要条件
    1. 互斥条件
    2. 不剥夺条件
    3. 请求且保持条件
    4. 环路等待条件

2.14 银行家算法，死锁定理

  1. 死锁定理

    当且仅当系统资源分配图是不可完全简化时，系统中有死锁。

    • 特别的，可以完全简化的时候，系统成功结束，没有简化出来的进程就是参与死锁的进程
  2. 银行家算法
    1. 安全状态

      安全状态是指分配的资源，满足每个进程的资源的最大需求，则每个进程顺利完成。

    2. 简单理解

      简单理解为：进程执行前，判断系统是否有足够的进程分配给该进程使得该进程能够成功执行完毕

3.2 地址重定位的概念，紧凑

3.2.1 概念

把用户程序装入内存时，对有关指令的逻辑地址部分的修改称为地址重地位。

3.2.2 紧凑

经过一段时间的使用之后，内存中存在大量的碎片，于是需要紧凑处理。移动各用户分区中的程序，使他们集中于内存的一端， 使碎片集中于另一端，从而将空闲的碎片连成一个较大的分区，供需求作业使用，为此，必须采用动态重定位技术。

3.3 页式，段式，段页式管理方式，数据结构，地址变换过程

3.3.1 分页式

  1. 概念

    通过查页表补全块号，构成实际地址

  2. 分配与回收

    位图法，链表法

  3. 地址重定位

    页内地址的长度由页大小决定, 逻辑地址由硬件分成的两部分页号p和页内地址w。例如：页大小1KB，则逻辑地址的低10位（$210 = 1KB$）为页内地址

    Table 1: 逻辑地址
    页号p(6位) 页内地址w（10位）
  4. 具体过程

3.3.2 段式

  1. 概念

    以段为基本单位分配内存，每一段必须分配连续的内存空间，单各段之间不要求连续。

    段号S 段内位移W
  2. 地址重定位

    检查段号，查段表，得出内存初始地址，与段内位移相加，得出物理地址

3.3.3 区别

  1. 页是信息的物理单位，段是信息的逻辑单位。页减少外部碎片。
  2. 分页式存储管理的作业地址空间是一维的，分段是二维的。
  3. 页的大小固定，段的长度不定。
  4. 分页的优点体现在内存空间的管理上，分段体现在地址空间的管理上

3.3.4 段页式

  1. 管理方式
    1. 先分段 作业地址空间进行段式管理
    2. 后分页 每段内在分成若干大小固定的页
    3. 物理块大小
    4. 分为三个部分
      段号(s) 段内页号(p) 页内位移(d)

4.3.2 索引节点

  1. 概念

    为了减少检索文件访问的物理块数，UNIX文件系统把文件目录项中的文件名和其他管理信息分开，后者单独组成定长的数据结构， 称为*索引节点*，也称i节点。文件目录项组成的目录文件与普通文件一样对待，存储在文件存储器中。

  2. i节点表

    文件存储器上的每一个文件，都有一个外存文件控制块（又称外存索引节点）i节点与之对应，这些i节点被集中存放在文件存储器的i节点区。 在使用的时候，内存会针对其建立一个内存i节点表（活动文件控制块或者活动i节点表）。

  3. 使用i节点表的流程

    使用一个文件，如果在活动i节点表中找不到其对应的i节点时，就申请一个内存i节点，把外存i节点的内容复制到内存i节点中，随之就使用 这个内存i节点来控制文件的读写。通常，最后一个用户关闭此文件后，内存i节点的内容被写到外存i节点中，然后，释放内存i节点。

4.3.3 树型目录结构

  1. 方便用户查找文件，又可以把不同类型和用途的文件分类
  2. 允许文件重名
  3. 利用树型结构关系，可以更方便的制定保护文件的存取权限，有利于文件的保护
  4. 不能直接支持文件或目录的共享
